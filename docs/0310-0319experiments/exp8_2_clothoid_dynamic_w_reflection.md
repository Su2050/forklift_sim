# Exp 8.2 实验结果与深度反思：Clothoid 曲线的极限与“最后半米”的视觉盲区

**日期**：2026-03-19
**实验分支**：`exp/vision_cnn/exp8_paper_native_refine` (Run: `exp8_2_clothoid_dynamic_w`)

## 1. 实验背景与核心配置

在之前的 Exp 8.1 中，我们通过释放绝对状态奖励的引力（去掉防爆常数 $c$）和降低惩罚权重，成功打破了 Agent 在 1.9m 处“匀速绕圈”的局部最优，使其敢于冲锋到托盘前 0.35m。然而，由于使用的是 Bézier 曲线，其在近端的曲率变化不够平滑，无法提供精确的偏航角引导，导致 Agent 在最后半米陷入“无效摩擦”，最终退缩。

为了彻底解决“末端对准”问题，我们在 Exp 8.2 中严格对齐了论文的核心思想，进行了以下重构：

1. **引入 Clothoid 曲线近似**：使用三次 Hermite 样条替换 Bézier 曲线，模拟 Clothoid 曲线曲率线性变化的特性，并**强制终点切线严格等于 0°（与托盘平行）**，为叉车铺设一条完美的奖励高速公路。
2. **动态权重增强末端引导**：在 `dist_front < 0.5m` 时，将偏航角对齐奖励 $r_{c\psi}$ 的权重放大 3 倍，告诉 Agent“越靠近，对准越重要”。
3. **引入 $L_{bound}$ 替代部分 $r_a$**：在奖励中加入 `r_bound = -||a||`，惩罚动作绝对值，防止方向盘打死，同时降低 $r_a$（动作变化惩罚）的权重，释放 Agent 的转向能力。
4. **严格对齐惩罚阈值**：确认 $r_p$（推盘惩罚）是基于速度的二值惩罚，而非位移惩罚；确认 $r_v$（超速惩罚）的存在。

## 2. 实验结果现象：从“坚守阵地”到“全面崩溃”

实验运行至 765 代，呈现出先扬后抑的戏剧性变化：

### 阶段一：完美的姿态控制与坚守 (Iter 0 ~ 500)
- **现象**：Clothoid 曲线的威力显现。Agent 停在距离托盘 **0.5m ~ 0.6m** 处，偏航角误差从 Exp 8.1 的 23° 大幅下降并稳定在 **13° ~ 20°** 之间。
- **行为**：它没有像 8.1 那样盲目贴脸硬推，而是保持在一个合理的距离进行“精准瞄准”。推盘位移极小（~0.2m），推盘惩罚极低。
- **结论**：Clothoid 曲线和动态权重成功“锚定”了 Agent，使其在长达 500 代的时间里顶住了“久攻不下”的压力，没有发生退缩。

### 阶段二：防线失守，全面崩溃 (Iter 500 ~ 765)
- **现象**：在坚持了 500 代的微调后，Agent 突然崩溃。
- **数据**：`dist_front` 退回到了 **1.07m**，偏航角飙升至 **63.1°**，推盘位移暴增至 **1.14m**，插入率跌至 **4.6%**。
- **行为**：它彻底放弃了在近端试探，再次变成了在 1.0m 外横着开的“巡逻车”。

## 3. 深度诊断：为什么 Clothoid 曲线也救不了它？

我们已经把奖励函数和轨迹生成优化到了数学上的极致，为什么它最终还是选择了放弃？

根本原因在于：**纯视觉基座（冻结的 ResNet34 + 256x256 分辨率）在“最后半米”的感知能力存在致命的物理盲区。**

为了验证这一点，我们编写了 `capture_insertion_sequence.py` 脚本，模拟了叉车从 0.5m 到完全插入的过程，并截取了 256x256 的实际网络输入图像。

**图像分析结论：**

**然而，在重新审视了 0.5m 处的网络输入图像后，我们发现了一个惊人的事实：**
**在 0.5m 处，托盘的视觉特征其实是非常明显的！** 托盘的孔洞、边缘在 224x224 的输入图像中占据了相当大的比例，特征极其清晰。

这就推翻了我们之前“分辨率不足导致看不清”的假设。

**真正的盲人摸象：冻结的 ResNet34 无法理解“几何语义”**
问题不在于“看不清”，而在于**“看不懂”**。
我们使用的是在 ImageNet（自然图像分类，如狗、猫、车）上预训练且**完全冻结**的 ResNet34。
- ImageNet 的特征提取器擅长回答“图片里是什么”（What），而不是“图片里的物体在什么精确坐标和角度”（Where & How）。
- 当叉车在 0.5m 处微调 5° 的偏航角时，图像确实发生了变化，但冻结的 ResNet34 提取出的 512 维特征向量，可能对这种“纯几何平移/旋转”极其不敏感。
- **结论**：Agent 不是“瞎子”，它是“几何认知障碍”。它看到了清晰的托盘，但它的视觉皮层（冻结的 Backbone）无法把这种视觉变化翻译成精确的相对位姿信号。

## 4. 结论与下一步战略转移 (Exp 9)

**Exp 8.2 的失败，标志着我们在“纯视觉 + 冻结 Backbone + 绝对状态奖励”这条路上的探索走到了尽头。**

论文中宣称的“仅靠冻结 ResNet 就能实现零样本 Sim2Real”，在我们的复现中被证明是不可行的。这可能是因为论文使用了双目摄像头、不同的图像裁剪策略，或者是未公开的辅助输入。

**战略转移：引入本体感觉辅助（Exp 9）**
既然冻结的视觉模型无法提供精确的几何语义，我们就必须给它一根“导盲犬的盲杖”。
在真实的工业应用中，叉车末端通常会安装传感器（如激光雷达、AprilTag）来获取精确的相对位姿。

**下一步计划**：
在 `env_cfg.py` 的观测空间 `proprio` 中，**加入相对位姿信息**（叉尖到托盘的相对坐标 $\Delta x, \Delta y$ 和相对偏航角 $\Delta \psi$）。
配合我们现在已经验证完美的 Clothoid 轨迹和奖励函数，Actor 网络将不再需要从模糊的图像特征中去“猜”相对位置，预计能在 500 代内实现完美收敛。
