# 实验 5 系列：有界 Shaping 与消除探索恐惧

## 1. 背景与动机 (从实验 4.1 的失败中吸取教训)
在实验 4.1（近场课程验证）中，我们修复了 `dist_center` 的几何计算 Bug，并把出生点拉近到 0.3m ~ 0.6m。我们期望在这个极近的距离下，Agent 能够顺理成章地插进去。
然而，**实验 4.1 依然失败了**。Agent 变成了“近场木头人”，停在距离托盘 1.1m 的地方不敢动。

**深刻的教训：**
1. **$1/x$ 奖励形式是万恶之源**：即使在近场，只要 Agent 姿态稍微对准，$1/rcd$ 和 $1/rc\psi$ 就会提供巨大的奖励。此时，如果它往前走，哪怕稍微偏离一点点轨迹，分母变大，奖励就会断崖式下跌。这形成了一个**“局部最优黑洞”**。
2. **论文的成功具有局限性**：原论文能在 $1/x$ 下成功，可能是因为其操作空间极小（1.8m），且可能有未公开的 Reward Clipping 机制。在我们的环境中，直接使用 $1/x$ 会导致数值极度不稳定和严重的 Reward Hacking。

## 2. 实验 5：稀疏终局奖励 + 轻量有界 Shaping
为了彻底解决“刷分黑洞”，我们决定重构正向奖励函数。

### 2.1 核心假设
将所有 Shaping 奖励（距离、轨迹）从无界的 $1/x$ 替换为有界的指数函数 $exp(-x/\sigma)$。这样任何一项 Shaping 的最高分都被死死卡在 1.0。同时，赋予终局奖励（`rg`）极高的权重。这样，**“完成任务”在数学上将成为唯一的全局最优解**。

### 2.2 修改内容
1. **替换公式**：
   - `r_dist = exp(-dist_center / 1.0)`
   - `r_traj_d = exp(-(d_traj / 0.3)^2)`
   - `r_traj_yaw = exp(-(yaw_traj_err_rad / 0.3)^2)`
2. **调整权重**：
   - `alpha_4` (rg, 到达奖励) 从 10.0 暴增至 **200.0**。
   - `alpha_8` (rini, 停滞惩罚) 降回 5.0（因为 Shaping 有界了，不需要那么大的惩罚来逼迫）。
3. **保持近场课程**：出生距离依然在 0.3m ~ 0.6m。

### 2.3 实验 5 运行现象：“探索瘫痪”
运行实验 5 后，发现 Agent **依然停在门外（1.1m）不敢进去**。
*   **原因分析**：虽然我们消除了 Shaping 的刷分漏洞，并挂上了 200 分的胡萝卜，但我们忽略了面前的“大棒”——**推盘惩罚 (`alpha_5 = 100.0`) 太高了**。
*   在随机探索初期，Agent 只要一靠近托盘，极大概率会发生碰撞，瞬间被扣 100 分。而它**从来没有**偶然触发过那 200 分的大奖。
*   因此，Critic 网络学到的经验是：“前方是雷区，停在原地虽然拿不到大奖，但至少不会被炸死”。这是一种典型的**“探索瘫痪 (Exploration Paralysis)”**或“习得性无助”。

## 3. 实验 5.1：消除探索恐惧 (当前正在进行)

### 3.1 核心假设
在课程学习的初期（破冰期），必须**大幅降低探索成本**，并**强行把 Agent “喂”到嘴边**，让它体验到一次 200 分的快感。等它脑子里刻下了“插入=巨大奖励”的潜意识后，再在后续课程中慢慢恢复难度和惩罚。

### 3.2 实施计划
1. **极大降低推盘惩罚（消除恐惧）**：
   - 将 `alpha_5` (rp) 从 100.0 暴降到 **1.0**。允许它在初期“粗鲁地”推盘插入。
2. **进一步收窄初始随机范围（消除无知）**：
   - 将 Stage A 的初始偏角从 `±10°` 降到 **`±2°`**。
   - 将横向偏差从 `±0.15m` 降到 **`±0.05m`**。
   - 目的：让它出生就正对托盘，只要踩一脚油门就能触发 `rg`。

### 3.3 预期目标
这不是为了训练出一个能用的最终模型，而是为了**打破认知盲区**。只要 `push_free_success_rate` 能够快速破零并上升，就说明“破冰”成功，随后我们将无缝进入 Stage B（扩大姿态难度）、Stage C（扩大距离难度）和 Stage D（恢复推盘惩罚）。

## 4. 实验 5.2 - 5.5b：打通最后半米的“课程退火”之路

在实验 5.1 中，虽然我们消除了探索恐惧，但 Agent 依然停在门外。随后的连贯实验揭示了强化学习在极近距离下的复杂博弈。

### 4.1 实验 5.2：修复“原地鬼畜”与轨迹奖励喧宾夺主
*   **发现**：Agent 学会了以 0.06m/s 的速度原地摇晃，既逃避了停滞惩罚，又吃满了轨迹对齐奖励。
*   **修复**：将停滞惩罚改为“向托盘的投影速度”，并移除近场轨迹奖励，改用绝对姿态奖励。
*   **结果**：Agent 被迫前进，但变成了“推土机”（顶着托盘走，不插进去）。

### 4.2 实验 5.3 & 5.4：放宽 rg 阈值，打破认知障碍 (重大突破)
*   **发现**：Agent 变成推土机是因为 `rg` 触发条件太苛刻（误差 < 0.1m），在随机探索下物理不可达，导致 Critic 永远不知道“插入”有大奖。
*   **行动 (Exp 5.4)**：将 `paper_rg_dist_thresh` 从 0.1 放宽到 **0.4m**（插进一半就算赢）。
*   **结果**：**巨大成功！** Agent 偶然触发大奖后瞬间开窍，`rg` 触发率飙升至 60% 以上，回合长度大幅缩短。

### 4.3 实验 5.5 vs 5.5b：步子迈大了容易扯着蛋 (课程退火的艺术)
*   **行动 (Exp 5.5)**：在 0.4m 阈值稳定后，我们试图直接将阈值收紧到 **0.2m**，并加载 Checkpoint 继续训练。
*   **结果 (Exp 5.5 失败)**：`rg` 瞬间跌回 0。Agent 发现无法轻易拿到大奖，于是退化回了“推土机”状态（托盘位移飙升至 0.95m，偏航角恶化至 15°）。
*   **反思**：从 0.4m（插 2/3）到 0.2m（几乎插到底），对姿态精度的要求呈指数级上升。直接跳跃导致了“灾难性遗忘”。
*   **修正 (Exp 5.5b 平滑退火)**：
    1. 将阈值退回到 **0.3m**。
    2. 将推盘惩罚 `alpha_5` 从 1.0 稍微提高到 **3.0**，抑制推土机行为。
    3. 重新加载 0.4m 时的 Checkpoint。
*   **最终结果 (Exp 5.5b 成功)**：Agent 完美适应了 0.3m 的新难度。`rg` 成功率稳定在 50%~60%，且**偏航角误差显著下降**（从 15° 降至 8°），托盘位移也被压制在 0.3m 左右。

### 4.4 核心经验沉淀：动态退火课程学习 (SOP)
实验 5.5b 的成功确立了我们在近场插入阶段的**标准操作流程 (SOP)**：
1. **宽容破冰**：先用极宽的阈值（如 0.4m）让 Agent 尝到甜头，激活 Critic 网络。
2. **平滑退火**：绝不能一次性收紧到底。必须以 0.1m 为步长（0.4 -> 0.3 -> 0.2 -> 0.1）逐步收紧。
3. **继承记忆**：每次收紧阈值，必须加载上一阶段稳定时的 Checkpoint，让“见过大奖”的 Critic 引导 Actor 提高精度。
4. **动态惩罚**：在收紧阈值的同时，可适当微调惩罚（如缓慢增加推盘惩罚），逼迫其放弃粗放动作，走向精细控制。
