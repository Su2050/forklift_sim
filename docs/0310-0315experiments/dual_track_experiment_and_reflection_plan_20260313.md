# 论文原生方法与 Master 基线双线复现计划 (去过度设计版 v4)

2. **"阈值必有算式支撑"**：任何几何阈值必须换算成实际插入深度，并与物理极限（当前托盘最大插入 0.938m）比对。
3. **"敬畏历史文档"**：开启新实验前，全局搜索相关的历史诊断报告，避免重复踩坑。

## 1. 核心反思与分支定位

### 1.1 为什么势函数在 Master 成功，但在 Vision 下容易失败？（分支 A 的价值）

Master 分支（15维状态输入）使用势函数（差分奖励）取得了成功。当我们切换到视觉输入（ResNet）时，势函数容易导致"原地鬼畜"等薅羊毛行为。其核心差异在于 **Actor 的学习难度（Vision Penalty）**：

- **Master (15D Actor)**：输入是精确的物理坐标。Actor 很容易建立"动作 -> 坐标变化 -> 势能差（奖励）"的精确映射。
- **Vision (Image Actor)**：输入是高维像素。Actor 需要建立"动作 -> 像素微小变化 -> 势能差（奖励）"的映射。这种映射极其困难且充满噪声。
- **分支 A 的定位**：作为**严格的控制变量基线**。如果分支 A 失败，我们就能确信瓶颈**完全在于视觉 Actor 的特征提取和映射能力**。

### 1.2 过去实验对"论文复现（分支 B）"的指导意义

- **Exp 5.9 (核弹奖励与微操)**：证明了 Agent 具备精确插入能力，但核弹权重(5000)导致 Value Function Loss 爆炸到十亿级别。
- **Exp 6.0/6.0b (温室花朵的泛化崩溃)**：即使微小泛化（±0.1m, ±5°）也导致性能骤降，证明"由窄到宽的课程学习"是死胡同。
- **叹息之墙 (物理极限)**：0.25m 和 0.15m 的 rg 阈值在物理上不可达（最大插入 0.938m），浪费了 5+ 个实验。
- **底层 Bug 修复**：`dist_center` 几何错误、`rini` 停滞惩罚、`_get_dones` 环境重置逻辑——这些修复是所有后续实验的基石。

### 1.3 论文成功的核心解法

论文不是靠复杂补丁，而是：
- **把任务限制在可见、可驾驶、可控制的范围内**
- **双目 ResNet34 预训练 + 2D 极简动作 + 每 episode 动态生成的参考轨迹奖励**
- **$r_g$ 只是二值辅助奖励{0,1}，真正引导学习的是三项连续轨迹奖励**

我们的偏离：单目相机（额外的深度歧义难度），需要用参考轨迹的持续引导来弥补。

## 2. 双线分支具体行动点

### 2.1 分支 A：Master 势函数基线 (控制变量法)

*   **目标**：验证在引入视觉 Backbone 后，原有的势函数奖励体系是否依然有效。
*   **核心配置**：
    *   **动作空间**：2D (Drive + Steer)（已确认，当前代码 `action_space=2`）。
    *   **视觉基座**：ResNet34 + ImageNet 预训练 + 全程冻结 (`freeze_backbone=True`)。
    *   **奖励函数**：完全回滚到 Master 分支的势函数逻辑（$\Phi_1, \Phi_2, \Phi_{ins}$ 的差分奖励）。
    *   **初始分布**：与 Master 分支保持完全一致的初始化范围。
*   **预期观察**：如果出现"薅羊毛"，重点分析 Critic 的 Value 估计是否准确，以及 Actor 的动作输出分布是否崩溃。

### 2.2 分支 B：论文原生方法复现 (去过度设计版)

*   **目标**：抛弃所有人为添加的"补丁"，用最纯粹的论文逻辑解决问题。
*   **核心配置**：
    *   **砍掉补丁**：删除 `w_align`、`w_band` 等软门控；废弃 $r_g$ 阈值退火；废弃空间衰减。
    *   **$r_g$ 回归辅助角色**：权重降回与 $\alpha_1 \sim \alpha_3$ 同量级（如 1~5），阈值固定为 **0.28m**（物理极限约束：最大插入 0.938m，0.28m 对应插入 0.92m）。
    *   **$r_{ini}$ 已对齐论文**：当前代码已正确实现 `proj_vel < 0.05 且 dist_front > 0.3` 的条件（已审计确认）。
    *   **参考轨迹奖励**：采用 **三次 Bézier + 直线（clothoid-lite）** 方案（参考 `experiment_3x` 文档 6.2~6.4 节）。
        *   每 episode 在 `_reset_idx` 中根据 spawn 位置动态生成。
        *   离散为 16~32 点 polyline 缓存，每步用 `torch.cdist` 做最近点投影。
        *   计算 $r_{cd}$（到轨迹距离）和 $r_{c\psi}$（与切线夹角）。
    *   **奖励形式**：优先使用 $\exp(-x/\sigma)$ 替代 $1/x$，避免数值爆炸（最终由阶段 0 验证决定）。
    *   **初始分布**：**先保守后放宽**，初始值基于 FOV 可见性验证结果确定（初步保守估计：距离 1.5~2.5m，横向 ±0.3~0.5m，偏航 ±10~15°）。
    *   **视觉基座**：ResNet34 + ImageNet 预训练 + 全程冻结。
    *   **Proprio**：8 维（已审计，全部为合法本体信息，无作弊）。

## 3. 实验执行顺序

1.  **创建公共基座与双分支**
    *   基于当前包含所有底层 Bug 修复的稳定代码（`exp/vision_cnn/generalization_stage1` HEAD），拉取公共基座分支 `exp/vision_cnn/dual_track_base`。
    *   从 `dual_track_base` 分化出 `exp/vision_cnn/branch_a_master_potential`。
    *   从 `dual_track_base` 分化出 `exp/vision_cnn/branch_b_paper_native`。

2.  **分支 A 验证 (快速试错)**
    *   在 `branch_a` 上恢复 Master 奖励和 2D 动作。
    *   运行基线训练，观察是否复现 Master 的成功，或记录视觉引入导致的特定失败模式。

3.  **分支 B 重构与验证 (主攻方向)**
    *   **阶段 0：数学与可见性验证（半天）**
        1.  实现 Bézier + 直线轨迹生成算法。
        2.  随机生成 100 个初始位置，画出轨迹，检查几何正确性和退化情况。
        3.  统计 FOV(90°) 内托盘可见比例，据此确定安全初始分布范围。
        4.  用模拟数据验证 $1/r_{cd}$ 和 $\exp(-r_{cd}/\sigma)$ 的数值范围，决定最终奖励形式。
    *   **阶段 1：代码清理与奖励重构**
        1.  砍掉门控、空间衰减、核弹奖励。
        2.  接入参考轨迹奖励（$r_{cd}$, $r_{c\psi}$）。
        3.  使用 debug_vis 在仿真中绘制轨迹进行视觉验证。
    *   **阶段 2：保守分布下基线训练**
        1.  在安全初始分布下启动端到端 PPO 训练。
        2.  前 200~300 iterations（约 2 小时）做快速冒烟判断。

4.  **深度复盘与超参调优**
    *   严格遵守"全链路反思"规则：如果失败，从物理环境 -> 观测 -> 价值函数 -> 奖励计算 逐一排查。
    *   重点调优 $\alpha_1, \alpha_2, \alpha_3$ 的相对权重（先固定惩罚项，只调正奖励比例）。
    *   如果保守分布下成功，逐步放宽初始分布范围。

## 5. 风险预判与缓解策略

| 风险 | 严重性 | 缓解策略 |
|------|--------|----------|
| 相机 FOV 盲区（托盘不在画面内） | **致命** | 阶段 0 验证 + 保守初始分布（确保托盘可见） |
| $1/x$ 数值爆炸导致 Critic 崩溃 | **致命** | 阶段 0 验证 + 优先使用 $\exp(-x/\sigma)$ |
| rg 阈值物理不可达 | **致命** | 固定为 0.28m（已有物理极限算式支撑） |
| "轨迹抱死"（贴轨迹不前进） | 严重 | 监控 $r_d$ 改善速度，必要时提高 $\alpha_1$ |
| "0.3m 停车"（近场停滞） | 严重 | 监控 $r_g$ 触发率，必要时微调 $\alpha_4$ |
| $\alpha$ 超参数维度爆炸 | 中等 | 先固定惩罚项，只调 $\alpha_1:\alpha_2:\alpha_3$ 比例 |
| 单目深度歧义 | 中等 | 参考轨迹引导弥补；若不足，后备方案引入 Depth Camera |
